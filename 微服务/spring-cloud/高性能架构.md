# 1.高性能架构

## 1.1 可用性比率计算

可用性比率用时间来计算：如果一年或者一个月

比如：一年 99.99%

可用时间：365 * 24 * 3600 * 99.99% 

不可用时间： 365 * 24 * 3600 * 0.01%   = 3153.6秒 < 1小时



单台机器不可用比率：1%

两台机器不可用比率：1% * 1%

## 1.2 可靠性

微服务里面的问题：

多个微服务链路调用

A -> B ->C   99% * 99%  * 99%  = 97%

A -> B ->C -> D  99% * 99%  * 99%  * 99% = 96%  逐渐降低



**总结：**

- 增加机器可以提供可用性；
- 增加服务调用会降低可靠性，同时降低了可用性；
