# 1. 查询优化

​	通过Explain 的rows字段预估查询可能得到的行，然后根据某一个值做判断，据经验当取出的数量超过表中的数据20%（固态硬盘这里不准确）,优化器就不会使用索引，而是进行全表扫描。

​	优化器不一定正确，因为row字段取出是预计的，不准确；所以有时候相信自己的判断使用force index强制使用索引查询。
